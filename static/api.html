API Reference

The Stix Magic REST API lets external apps read and manage sticker packs created through the Telegram bot.

Base URL

https://stixmagic.com/api

Version

Current version: 1.0. The API version is returned in every response header as X-API-Version.

Authentication

All endpoints except /api/health require an API key.

Pass your key via the X-API-Key request header or as an api_key query parameter.

Header (recommended)

curl https://stixmagic.com/api/stats \
  -H "X-API-Key: your_api_key_here"

Query param

curl "https://stixmagic.com/api/stats?api_key=your_api_key_here"

Response Format

All responses use a consistent JSON envelope.

Success

{
  "ok": true,
  "data": { /* payload */ }
}

Error

{
  "ok": false,
  "error": {
    "message": "Human-readable description",
    "code": "machine_readable_code"
  }
}

HTTP status codes

CodeMeaning
200Success
400Bad request (invalid params)
401Unauthorized (missing or wrong API key)
404Resource not found
405Method not allowed
500Server error

Pagination

List endpoints support cursor-based pagination via query params.

ParamTypeDefaultDescription
page integer 1 Page number (1-indexed)
limit integer 20 Items per page (max 100)

Pagination in response

{
  "ok": true,
  "data": [ /* items */ ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 47,
    "pages": 3
  }
}

Response

  "ok": true,
  "data": {
    "users": 128,
    "packs": 512,
    "users_with_settings": 74
  }

Search Packs

GET /api/search?q={query} AUTH

Search packs by title or Telegram pack name. Supports pagination.

Query params

ParamTypeDescription
q string required Search query (min 2 chars)
page integer optional Page number
limit integer optional Items per page (max 100)

Response

{
  "ok": true,
  "data": [
    {
      "user_id": 123456789,
      "name": "stix_123456789_abcde_by_stixmagicbot",
      "title": "My Cool Pack",
      "link": "https://t.me/addstickers/stix_123456789_abcde_by_stixmagicbot"
    }
  ],
  "query": "cool",
  "pagination": { "page": 1, "limit": 20, "total": 1, "pages": 1 }
}

List User Packs

GET /api/packs/{user_id} AUTH

Returns all sticker packs belonging to a Telegram user. Supports pagination.

Path params

ParamTypeDescription

Response

{

Delete Pack

DELETE /api/packs/{user_id}/{pack_name} AUTH

Removes a pack from the Stix Magic database. The Telegram sticker pack itself is unaffected.

Response

{
  "ok": true,
  "data": {
    "deleted": true,
    "name": "stix_123456789_abcde_by_stixmagicbot"
  }
}

Get Settings

GET /api/settings/{user_id} AUTH

Returns a user's current bot settings.

Response

{
  "ok": true,
  "data": {
    "user_id": 123456789,
    "mask_inverted": false
  }
}

Update Settings

PATCH /api/settings/{user_id} AUTH

Updates a user's settings. Send only the fields you want to change.

Request body

{
  "mask_inverted": true
}

Example

curl -X PATCH https://stixmagic.com/api/settings/123456789 \
  -H "X-API-Key: your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"mask_inverted": true}'

Response

{
  "ok": true,
  "data": {
    "user_id": 123456789,
    "mask_inverted": true
  }
}
